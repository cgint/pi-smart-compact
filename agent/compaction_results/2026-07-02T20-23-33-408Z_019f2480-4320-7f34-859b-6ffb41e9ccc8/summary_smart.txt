## What Was Done
Conducted comprehensive research and synthesis to design a "prompt-only" compaction mechanism for Pi sessions, aiming to replace Pi's built-in summarization with a higher-signal, structured output.
- Gathered baseline data on Pi's existing `compaction.js` implementation (triggers, cut points, prompt structures).
- Analyzed trace-v2 session logs to identify preservation priorities (decisions with rationale, evidence links, episode boundaries).
- Reviewed academic literature (ACE, GAM, STRATUS) for context engineering best practices (evolving playbooks, incremental updates).
- Investigated technical feasibility of implementing a custom compaction extension, specifically focusing on accessing the LLM model and authentication context within Pi's extension hooks.

## Key Findings / Observations
- **Pi Baseline Limitations**: Pi's default compaction uses a generic system prompt ("You are a context summarization assistant"), truncates tool results to 2000 chars, and lacks strategic prioritization guidance.
- **Preservation Priorities**: High-value compaction must preserve: structured sections (Goal, Decisions, Next Steps), evidence-linked decisions (96.4% of trace-v2 messages have rationale), file operations, and the last ~20K tokens verbatim.
- **Extension Auth Barrier**: Initial attempts to call the LLM via `completeSimple()` failed because `currentModel` is null during auto-compaction and `modelRegistry` auth is inaccessible in standard extension contexts.
- **Viable Path Identified**: The `session_before_compact` hook combined with `ctx.modelRegistry` (available via `runner.bindCore`) allows resolving API keys and calling the LLM correctly, matching Pi's internal extension patterns.
- **North Star Alignment**: The primary goal is a single-prompt solution that outputs a dense, structured continuation context without relying on external side-car files or lookup tables at runtime.

## Key Decisions
- **[Approach: Prompt-Only First Milestone]**: The initial deliverable is a refined compaction prompt (`DRAFT_COMPACTION_PROMPT.md`) rather than complex side-storage mechanisms. Rationale: Simpler to implement and validate; aligns with the primary goal of improving signal-to-noise ratio.
- **[Implementation: Use `ctx.modelRegistry`](https://github.com/...])**: To bypass auth issues, the extension must use `ctx.modelRegistry` to resolve credentials for the current session model, rather than attempting to derive them manually. Rationale: Verified against Pi v0.80.3 source; this is the only working method to access authenticated LLM calls within the compaction hook.
- **[Content Strategy: Evidence-Linked Decisions]**: The compaction prompt must enforce a structure where decisions are paired with rationale and evidence references. Rationale: Trace-v2 analysis shows this is the highest-value signal for agent continuity.
