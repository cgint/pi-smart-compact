## Key Findings / Observations
- **Pi Compaction Limitations:** Pi's default compaction truncates tool results to 2000 chars, uses a generic system prompt lacking strategic guidance, and relies on a rigid template that may lose nuanced decision rationale.
- **Extension Auth Barrier:** Direct LLM calls from extensions fail due to inaccessible API keys/auth layers. The viable solution requires using `ctx.modelRegistry` exposed in the `session_before_compact` hook context.
- **High-Signal Preservation:** Trace-v2 data shows 96.4% of messages contain rationale/evidence. Effective compaction must preserve these "why" elements rather than just summarizing "what" happened.
- **Structural Integrity:** Successful compaction requires incremental merging of previous summaries to prevent information erosion over multiple compaction cycles.

## Key Decisions
- **[Adopt Prompt-Only Milestone]**: The first milestone focuses exclusively on improving the compaction prompt strategy without relying on external storage or lookup systems, ensuring the compacted output carries all necessary continuation context.
- **[Use `ctx.modelRegistry` for Extension]**: To bypass authentication failures, the extension implementation must utilize `ctx.modelRegistry` provided in the `session_before_compact` event context to resolve API keys and make LLM calls.
- **[Preserve Decision Rationale & Evidence]**: The compaction prompt is designed to prioritize the preservation of decisions with their underlying rationale and evidence references, treating them as higher value than raw narrative history.

## Current State
