## Key Findings / Observations
- **Pi's Baseline Limitations:** Pi's built-in compaction uses a generic "summarization assistant" prompt, lacks strategic prioritization, and truncates tool results before summarization, leading to information loss. However, it correctly uses incremental merging and preserves file operations.
- **Signal Density:** Trace-v2 data shows that preserving "evidence-linked decisions" (intent + rationale + evidence) is critical, as this represents the highest-signal portion of a session.
- **Extension Feasibility:** Direct LLM calls from extensions fail due to auth isolation. The viable path is using `session_before_compact` combined with `ctx.modelRegistry` to resolve API keys and make an LLM call, mirroring Pi's internal auth flow.
- **Prompt Design:** Effective compaction requires separating "active state" (goals, blockers, next steps) from "historical context" (evidence, traces), potentially using context pointers for the latter to save tokens.
- **Research Alignment:** Academic sources (ACE) support "evolving playbooks" and "structured incremental updates" over flat summarization.

## Key Decisions
- **[Milestone Definition]**: The first milestone is a **prompt-only** compaction mechanism delivered via an extension hook, avoiding side-storage or external retrieval systems initially.
  - *Rationale*: Aligns with NORTH-STAR to prove value of smarter prompting before adding infrastructure complexity.
- **[Implementation Strategy]**: Use `session_before_compact` hook with `ctx.modelRegistry` to handle authentication.
  - *Rationale*: Previous attempts to bypass auth failed; this leverages Pi's internal security model correctly.
- **[Compaction Logic]**: Adopt a "preserve recent verbatim, summarize historical" strategy, keeping the last ~20K tokens intact.
  - *Rationale*: Matches Pi's default `keepRecentTokens` setting and ensures the immediate context remains uncorrupted.
- **[Output Structure]**: The draft prompt enforces specific sections: Big Picture, Work Stream Evolution, Constraints, Progress, Key Decisions (with rationale/evidence), Next Steps, Critical Context, and File Operations.
  - *Rationale*: Based on analysis of successful trace patterns and ACE principles to maximize information density per token.
