## Goal
Develop and validate a "prompt-only" compaction mechanism for Pi sessions that serves as a drop-in replacement for Pi's built-in compaction. The primary objective is to intercept the `session_before_compact` hook, generate a high-signal, minimal continuation context via an LLM call (using a custom prompt defined in `findings/DRAFT_COMPACTION_PROMPT.md`), and return it to Pi, ensuring graceful fallback if the LLM call fails.

## Constraints & Preferences
- **Prompt-Only Milestone:** The first milestone must rely solely on a prompt-based transformation; no side-storage, lookup tables, or external retrieval systems during the compaction act.
- **Graceful Degradation:** If the custom LLM call fails, the extension must automatically fall back to Pi's native compaction to avoid session disruption.
- **Auth Handling:** Must use Pi's internal `ctx.modelRegistry` for authentication; cannot construct external auth or bypass Pi's security layers.
- **Output Structure:** The generated compaction must follow a specific structured format (Goal, Constraints, Progress, Decisions, Next Steps, Critical Context) to ensure readability for subsequent agents.
- **Preservation Rules:** Must preserve exact file paths, function names, error messages, and the last ~20K tokens of conversation verbatim.

## Key Decisions
- **[Architecture Change]**: Switched from attempting to inject custom prompts into Pi's internal JS engine to building a standalone Node.js extension that intercepts the `session_before_compact` event.
- **[Auth Resolution]**: Decided to use `ctx.modelRegistry.getApiKeyAndHeaders(model)` provided by Pi's runner context rather than deriving auth manually, which previously caused failures.
- **[Compaction Strategy]**: Adopted a "structured summary" approach inspired by Pi's existing template but enhanced with "evidence-linked decisions" and "context pointers" for lower-priority details, based on ACE paper principles.
- **[Fallback Mechanism]**: Explicitly implemented a try-catch block around the LLM call to revert to Pi's default behavior on failure, ensuring stability.
