## Goal
Create a "Prompt-Only" compaction mechanism for Pi sessions that replaces Pi's built-in compaction. The primary objective is to distill an overloaded agentic session into a minimal, high-signal continuation context using only a custom LLM prompt, without relying on side-storage, lookup tables, or external retrieval systems during runtime.

## Constraints & Preferences
- **First Milestone Constraint**: Must be prompt-only. No additional snippets, side files, lookup tables, or external retrieval created during compaction. The compaction output itself must carry the essential continuation context.
- **Output Requirements**: Minimum characters with maximum information. Preserve big-picture objective, current phase, current-state detail, decisions, and next steps. Low noise/stale history.
- **Graceful Degradation**: If LLM call fails, fall back to Pi's built-in compaction.
- **Installation**: `pipa install -c .` (extension installs automatically).
- **Configuration**: Edit `findings/DRAFT_COMPACTION_PROMPT.md` to change behavior.

## Key Decisions
- **[Prompt-Only Strategy]**: Chose to replace compaction entirely via a single custom prompt rather than hybrid approaches, to minimize runtime overhead and complexity for the first milestone.
- **[Auth Resolution]**: Used `ctx.modelRegistry` provided by Pi's runner to resolve API keys/models dynamically, avoiding hardcoded credentials or inaccessible internal states.
- **[Structure Preservation]**: Retained Pi's structured output format (Goal, Constraints, Progress, etc.) but enhanced it with "Work Stream Evolution" and stricter evidence-linking rules based on trace-v2 findings.
- **[Recent Context Retention]**: Preserved last ~20K tokens verbatim, summarizing only older history, consistent with Pi's native behavior but enforced via prompt instructions.
